# Resumo dos principais tópicos do PMBOK

## 📚 Estrutura básica
| Camada | Descrição |
|--------|-----------|
| **Grupos de Processos** | Sequência lógica de ações em um projeto. São 5 grupos. |
| **Áreas de Conhecimento** | Conjunto de competências necessárias para gerir cada aspecto do projeto. São 10 áreas. |
| **Processos** | 49 processos detalhados que ligam grupos e áreas (ex.: *Planejar o Cronograma*, *Controlar os Riscos*). |

## 🔄 Grupos de Processos (5)

1. **Iniciação** – Definir o projeto ou fase.  
   - *Desenvolver o Termo de Abertura*  
   - *Identificar as Partes Interessadas*

2. **Planejamento** – Estabelecer o escopo, objetivos, e curso de ação.  
   - Ex.: *Desenvolver o Plano de Gerenciamento do Projeto*, *Planejar o Cronograma*, *Planejar o Orçamento*, *Planejar a Qualidade*, *Planejar os Riscos*, *Planejar as Aquisições*, etc.

3. **Execução** – Realizar o trabalho definido no plano.  
   - *Dirigir e Gerenciar o Trabalho do Projeto*  
   - *Gerenciar o Engajamento das Partes Interessadas*  
   - *Desenvolver a Equipe*  
   - *Gerenciar a Comunicação*, etc.

4. **Monitoramento e Controle** – Acompanhar o progresso e fazer ajustes.  
   - *Monitorar o Trabalho do Projeto*  
   - *Controlar o Cronograma*  
   - *Controlar o Orçamento*  
   - *Controlar as Mudanças*, etc.

5. **Encerramento** – Formalizar a aceitação e finalizar o projeto/fase.  
   - *Encerrar o Projeto ou Fase*  
   - *Encerrar as Aquisições*

## 📂 Áreas de Conhecimento (10)

| Área | Principais Processos (exemplos) |
|------|---------------------------------|
| **Integração** | Desenvolver o Termo de Abertura, Gerenciar o Conhecimento do Projeto, Encerrar o Projeto |
| **Escopo** | Planejar o Gerenciamento do Escopo, Definir o Escopo, Validar o Escopo, Controlar o Escopo |
| **Cronograma** | Definir Atividades, Estimar Durações, Desenvolver o Cronograma, Controlar o Cronograma |
| **Custos** | Estimar Custos, Determinar o Orçamento, Controlar o Orçamento |
| **Qualidade** | Planejar a Qualidade, Gerenciar a Qualidade, Controlar a Qualidade |
| **Recursos** | Planejar o Gerenciamento dos Recursos, Estimar Recursos, Adquirir Recursos, Desenvolver a Equipe, Gerenciar a Equipe, Controlar os Recursos |
| **Comunicações** | Planejar as Comunicações, Gerenciar as Comunicações, Monitorar as Comunicações |
| **Riscos** | Planejar o Gerenciamento dos Riscos, Identificar Riscos, Analisar Qualitativamente, Analisar Quantitativamente, Planejar Respostas, Monitorar Riscos |
| **Aquisições** | Planejar as Aquisições, Conduzir as Aquisições, Controlar as Aquisições |
| **Partes Interessadas** | Identificar Partes Interessadas, Planejar o Engajamento, Gerenciar o Engajamento, Monitorar o Engajamento |

## 🛠️ Ferramentas e Artefatos-chave
- **EAP (Estrutura Analítica do Projeto)** – Decomposição hierárquica do escopo.  
- **Cronograma (Diagrama de Gantt, CPM, PERT)** – Visualização de atividades e dependências.  
- **Matriz de Responsabilidades (RACI)** – Define quem é Responsável, Aprovador, Consultado e Informado.  
- **Registro de Riscos** – Lista de riscos, suas probabilidades, impactos e planos de resposta.  
- **Registro de Mudanças** – Controle de solicitações de mudança e decisões.  
- **Relatórios de Status** – Indicadores de desempenho (valor agregado, índice de desempenho de prazo/custo).

> **Observação do professor (70 anos de experiência)**: ao estudar o PMBOK, foque não só na memorização dos 49 processos, mas na compreensão de **por que** cada processo existe. A interdependência entre grupos e áreas de conhecimento é o que traz a flexibilidade necessária para adaptar o framework a projetos reais, especialmente em ambientes ágeis e híbridos.
